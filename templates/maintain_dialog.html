{# Maintain dialog: tabbed modal exposing long-running maintenance operations (writeback, sync-metadata, verify) to the web UI. Each tab is a small form that POSTs to /api/maintain/ and hands the returned job_id off to window.makiJob.watch() — the same toast pipeline the import dialog uses, so live progress and re-attach come for free. The tabbed shell is the only Maintain-specific UI; per-tab forms are intentionally minimal because most users will run these commands rarely. Reuses .import-overlay/.import-modal styling so the dialog visually matches the existing Import / batch confirmation modals without inventing a separate skin. #}

{# The dialog has three top-level phases that swap in/out of the modal body, mirroring the import dialog's lifecycle: - "forms" — the tabbed forms (initial + after Close on a finished job) - "progress" — live progress for a running job - "result" — terminal summary + Close button The tabs container and per-tab panes all live inside #maintain-forms so we can hide the whole forms phase by toggling one element. #}
{# ── Writeback ───────────────────────────────────────────── #}

Flush pending metadata edits (rating, label, description, tags) from the catalog to .xmp recipe files on disk. Always works regardless of [writeback] enabled.

{# ── Sync metadata ───────────────────────────────────────── #}

Bidirectional XMP ↔ catalog sync. Inbound: external .xmp changes are pulled into the catalog. Outbound: pending catalog edits are flushed to .xmp when no conflict exists (a recipe changed on disk AND in MAKI).

{# ── Verify ──────────────────────────────────────────────── #}

Re-hash media + recipe files on disk and compare against the catalog's stored content hashes. Surfaces silent corruption (bit-rot, accidental edits, sync glitches) and missing files.

{# ── Generate previews ──────────────────────────────────── #}

(Re-)generate the asset thumbnails the browse grid shows. Useful after fix-roles --apply moved a different variant into the best-preview slot, or to add 2560px smart previews to assets that only have the regular thumbnail.

{# ── Sync (file layer) ──────────────────────────────────── #}

Reconcile a volume's catalog locations with what's actually on disk: detect moved, modified, new, and missing files. Use Apply to update the catalog; use Remove stale to drop locations for files that are confirmed missing.

{# ── Refresh (re-read source metadata) ──────────────────── #}

Re-read metadata from .xmp sidecar/recipe files that changed on disk since the last scan. With media XMP on, the embedded XMP block in JPEG/TIFF is also re-extracted. Use after editing in Lightroom / Capture One while MAKI was offline.

{# ── Cleanup ────────────────────────────────────────────── #}

Remove orphaned records and files: stale catalog locations whose files no longer exist; locationless variants; orphaned assets, previews, embeddings, and face crops left behind by deletes or relocations.
